## 学习笔记

### 2026-06-18

**完成内容：**
- 环境搭建：Go 1.26.4
- 完成第1阶段：Go 基础语法（1.1-1.8 全部完成）

**关键收获：**
1. Go 程序必须有 `package main` 和 `func main()`
2. 包（package）= 一个目录下的 .go 文件集合
3. 同一个包内函数可以直接调用，不需要 import
4. 模块（module）= 项目根目录，用 `go.mod` 管理
5. 运行一个包要用 `go run ./目录名/`，而不是单个文件
6. `:=` 声明+赋值，`=` 赋值（变量必须已声明）
7. 常量用 `const`，`iota` 自动生成递增值
8. 切片 `[]int` 是动态数组，类似 Python list
9. Map 是键值对，类似 Python dict
10. 多返回值按顺序赋值，`_` 丢弃不需要的值
11. Go 没有三元表达式，必须用 if/else
12. 大写开头 = 公开，小写开头 = 私有

**遇到的问题：**
- 只运行单个文件会找不到同包的函数 → 解决：运行整个目录
- 没有 go.mod 会报错 → 解决：`go mod init`
- 目录名不能用中文 → 解决：用英文命名

### 2026-06-20

**完成内容：**
- 完成第2阶段：Go 核心特性（2.1-2.5 全部完成）

**关键收获：**
1. 指针：&取地址，*解引用，new创建指针
2. Struct：自定义数据类型，类似Python class，方法写在外面
3. Interface：定义行为规范，隐式实现，解耦
4. 错误处理：error接口，errors.New，fmt.Errorf
5. defer/panic/recover：defer清理，panic崩溃，recover恢复

**核心理解：**
- 指针就是变量的内存地址，通过地址可以直接修改原变量
- Struct 是数据容器，Interface 是行为规范
- Interface 的核心作用：只关心"能做什么"，不关心"是什么"
- Go 的错误处理：返回 error，必须显式处理
- defer 保证清理工作一定执行，panic 是意外错误，recover 捕获 panic
