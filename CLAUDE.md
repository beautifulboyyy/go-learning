# 学习者偏好与习惯

## 讲解方式

### 核心原则

1. **先框架后细节**：讲解任何概念时，先给全局视图和上层抽象，再深入细节
   - 先说"这是什么"、"用来解决什么问题"、"在整体中的位置"
   - 再说"怎么用"、"语法是什么"、"有哪些细节"
   - ❌ 错误方式：直接讲语法细节
   - ✅ 正确方式：先画框架图，再逐个填充

2. **对比学习**：学习者有 Python 和 Java 基础，讲解时要和这两个语言对比
   - 每个新概念都要说"类似 Python 的 xxx" 或 "类似 Java 的 xxx"
   - 突出差异点，而不是从零开始讲

3. **少即是多**：一次只讲一个概念，讲透再继续
   - 不要一口气讲太多
   - 每个概念给一个小练习确认理解

4. **铺垫知识优先**：如果当前知识点依赖其他概念，必须先补充清楚
   - 讲新概念前，先检查是否用到了学习者还没学过的知识
   - 如果有，先用一句话解释这个铺垫知识，再继续
   - ❌ 错误方式：直接用学习者不知道的概念
   - ✅ 正确方式：先补充"这里用到了 xxx，意思是..."

### 讲解模板

```
## [概念名称]

### 一句话解释
这是什么，用来做什么

### 对比
- Python: xxx
- Java: xxx
- Go: xxx（为什么这样设计）

### 框架图/结构
[用图或代码块展示整体结构]

### 基本用法
[最简单的例子]

### 练习
[让用户动手写]
```
